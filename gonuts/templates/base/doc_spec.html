Nut specification

Nut metadata is stored in two places. Standard Go entities like package name and import path are accessible via standard package go/build. The rest is stored in file called nut.json, which can be created and updated with nut generate command. This document defines its format. 
{
  "Version": "0.3.0",
  "Vendor": "AlekSi",
  "Authors": [
    {
      "FullName": "Alexey Palazhchenko",
      "Email": "alexey.palazhchenko@gmail.com"
    }
  ],
  "ExtraFiles": [
    "CHANGELOG",
    "CONTRIBUTING.md",
    "LICENSE",
    "README.md"
  ],
  "Homepage": "http://www.gonuts.io/"
}
  • Version defines nut version according to specification.
  • Vendor FIXME
  • Author defines nut authors and maintainers. Each array element is a FIXME
  • ExtraFiles defines an array of files which are a part of nut, but do not contain code and therefore are not a part of Go package.
  • Homepage defines nut's homepage in a form of HTTP/HTTPS URL.
